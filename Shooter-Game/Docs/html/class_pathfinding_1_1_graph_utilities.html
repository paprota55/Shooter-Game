Detailed Description

Contains utility methods for getting useful information out of graph.

This class works a lot with the Pathfinding.GraphNode class, a useful function to get nodes is AstarPath.GetNearest.

Finds all contours of a collection of nodes in a grid graph.

var grid = AstarPath.active.data.gridGraph;

// Find all contours in the graph and draw them using debug lines GraphUtilities.GetContours(grid, vertices => { for (int i = 0; i < vertices.Length; i++) { Debug.DrawLine(vertices[i], vertices[(i+1)vertices.Length], Color.red, 4); } }, 0);

Parameters
gridThe grid to find the contours of

static List<Vector3> Pathfinding.GraphUtilities.GetContours ( NavGraph  graph)
static

Convenience method to get a list of all segments of the contours of a graph.

Returns: A list of segments. Every 2 elements form a line segment. The first segment is (result[0], result[1]), the second one is (result[2], result[3]) etc. The line segments are oriented so that the navmesh is on the right side of the segments when seen from above.

This method works for navmesh, recast, grid graphs and layered grid graphs. For other graph types it will return an empty list.

If you need more information about how the contours are connected you can take a look at the other variants of this method.

// Get the first graph var navmesh = AstarPath.active.graphs[0];

// Get all contours of the graph (works for grid, navmesh and recast graphs) var segments = GraphUtilities.GetContours(navmesh);

// Every 2 elements form a line segment. The first segment is (segments[0], segments[1]), the second one is (segments[2], segments[3]) etc. // The line segments are oriented so that the navmesh is on the right side of the segments when seen from above. for (int i = 0; i < segments.Count; i += 2) { var start = segments[i]; var end = segments[i+1]; Debug.DrawLine(start, end, Color.red, 3); }
